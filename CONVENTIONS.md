# 约定与规范

本文档记录项目的写作约定、内容规范和组织原则，后续所有内容补充应遵守。

## 内容规范

### 语言
- 中文为主，英文术语保留原文
- 首次出现的专业术语格式：`中文名（English Term）`
- 引用格式参照学术论文，使用 Markdown 脚注或末尾参考文献列表

### 写作风格
- 面向有软件工程背景的读者，不需要解释基础编程概念
- 可以假设读者了解：HTTP/API、设计模式、分布式系统基础、基本的 ML 概念
- 不可以假设读者了解：Agent 领域专有概念（需要解释）
- 保持工程师视角，理论与实践并重

### 深度控制
- LLM 之前的历史：简略介绍，关注与 LLM Agent 有启发的理论基础
- LLM Agent 时期：重点详述，框架/产品/模式均需深入
- 宏观叙事为主，具体 case 用几句话总结即可

### 引用格式
```
[作者/组织, 年份] 标题. 来源/链接
```
示例：
```
[Yao et al., 2023] ReAct: Synergizing Reasoning and Acting in Language Models. ICLR 2023.
[OpenAI, 2023] GPT-4 Technical Report. https://arxiv.org/abs/2303.08774
```

## 组织原则

### 目录结构
- 按主题树状组织，不按时间线铺平
- 每个目录下有 `_index.md` 作为该章节的概览和导航
- 具体内容文件用有意义的英文短名，如 `react-pattern.md`

### 文件命名
- 目录名：小写英文，连字符分隔，如 `multi-agent/`
- 文件名：小写英文，连字符分隔，如 `tool-use-patterns.md`
- 索引文件统一命名：`_index.md`

### 内容冗余
- 允许不同章节对同一概念有不同角度的描述
- 不同框架下概念划分可能重叠，均予以收录
- 通过交叉引用（相对路径链接）关联相关内容

### 版本与迭代
- 重大结构调整记录在 `CHANGELOG.md`
- 日常内容补充无需记录
- 每个文件头部可选标注：`<!-- last updated: YYYY-MM-DD -->`

## 边界说明

- 无明确排除内容，学术论文、商业产品、代码教程均可收录
- 如果某个知识点有多个来源的不同说法，都收录并标注来源
- 工程实践和理论分析并重
