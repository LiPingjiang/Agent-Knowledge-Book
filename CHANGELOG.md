# 迭代记录

记录项目重大决策和结构变更，便于追溯思路演变。

## 2025-06 · 项目启动

### 背景
构建一部 Agent 知识大全，系统性整理 Agent 领域从历史到前沿的完整知识体系。

### 决策记录

**项目定位**
- 受众：软件研发工程师，转型 Agent 研发方向
- 载体：纯 Markdown 文档树（知识底座），后续再研究可视化和交互
- 维护：手动补充
- 语言：中英双语，中文为主，历史资料保留引用

**内容范围**
- Part I · Agent 史学：从 1950s 至今的 Agent 发展历史
  - LLM 之前：简略介绍，重点关注对 LLM Agent 有启发的理论基础
  - LLM Agent 时期：重点详述各种 LLM-based Agent 的发展
  - 以宏观叙事为主，具体 case 几句话总结
- Part II · Agent 技术体系：当下先进技术的系统拆解
  - 框架/产品：尽可能全面，进行分类
  - 内部模块：细致拆解，允许冗余，各家概念都收录
  - 覆盖 Multi-Agent 协作模式
  - 覆盖评测体系

**组织原则**
- 按主题树状组织
- 允许不同框架概念重叠
- 无明确排除内容
- 不分期，整体规划后逐步充实
