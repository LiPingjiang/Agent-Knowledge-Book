# 写作规范与约定

本文件供写书时参考，不面向最终读者。

## 语言约定

- 正文以中文撰写
- 英文术语首次出现时标注原文，后续可直接使用
- 代码示例、配置、CLI 命令保留英文

## 文件组织

- 每章一个目录，目录下 `_index.md` 为该章导读页
- 正文文件以英文短横线命名（kebab-case），如 `bdi-architecture.md`
- README.md 是面向读者的书籍首页，不暴露文件名、目录结构等技术细节
- 附录放在 `docs/appendix/` 下

## 内容标准

- 每篇正文章节目标 150-300 行，包含理论、代码示例、对比表格
- 引用的论文标注 arXiv ID 或出处
- 数据、统计数字须注明来源和时间
- 区分已确认事实与业界推测

## Deep Research 集成流程

1. 收到 Deep Research 报告后，先做质量评估（8 分制）
2. 与现有章节逐一对比，找出增量价值
3. 空白章节直接填充，已有章节仅补充新增内容
4. 集成后在 `docs/appendix/deep-research-review.md` 记录评审日志

## README 编写原则

- README 是书的"封面 + 目录"，面向读者
- 使用人类可读的章节标题作为链接文字
- 不暴露原始文件名（用户不关心 .md 后缀和 kebab-case 命名）
- 不放置项目元信息、写作规范等内部内容
- 保持简洁优雅，像一本书的目录页
