# 认知架构：SOAR 与 ACT-R

> SOAR 和 ACT-R 是两个历史最悠久、影响最深远的认知架构，它们试图从计算层面模拟人类的通用智能机制，并在 2024-2026 年为 LLM Agent 的推理设计提供了重要的理论指导。

## 概述

与 BDI 模型的高层哲学抽象不同，SOAR 和 ACT-R 是具体的、可运行的计算架构。它们都起源于 1980 年代的认知科学研究，目标是构建一个"统一认知理论"（Unified Theory of Cognition）——用单一架构解释人类认知的广泛现象，包括问题解决、学习、记忆、语言理解和决策。

两者在认知科学领域积累了数十年的实验验证，产生了数千篇学术论文，并被应用于从军事训练模拟到人机交互设计等多个领域。

## SOAR (State, Operator, And Result)

### 历史背景

SOAR 由 Allen Newell、John Laird 和 Paul Rosenbloom 在 1983 年于卡内基梅隆大学创建。Newell 将其视为实现"统一认知理论"的核心尝试——他在 1990 年的同名著作中详细阐述了这一愿景。SOAR 至今仍在密歇根大学的 John Laird 团队维护和发展。

### 核心机制

SOAR 将所有认知任务统一建模为在**问题空间**（problem space）中的搜索过程：

**状态（State）**：表示问题求解过程中的当前情境，包括目标、已知信息和中间结果。

**算子（Operator）**：表示可以施加于状态的变换操作。Agent 通过选择和应用算子来从一个状态转移到另一个状态。

**目标（Goal）**：表示 Agent 期望达到的状态。问题求解就是从初始状态出发，通过一系列算子应用，到达目标状态。

### SOAR 的决策循环

SOAR 的运行核心是一个快速的决策循环（约 50ms 一个周期，对应人类认知的基本时间单元）：

```
输入阶段 → 提议阶段 → 决策阶段 → 应用阶段 → 输出阶段
   ↑                                              ↓
   └──────────────────────────────────────────────┘
```

1. **输入阶段（Input）**：从感知缓冲区读取外部信息
2. **提议阶段（Propose）**：产生式规则（production rules）触发，提议可能的算子
3. **决策阶段（Decision）**：根据偏好选择一个算子
4. **应用阶段（Apply）**：将选中的算子应用于当前状态
5. **输出阶段（Output）**：向执行系统发送动作命令

### 僵局与子目标机制

SOAR 最独特的机制是其对**僵局**（impasse）的处理。当系统在决策阶段无法确定下一步该做什么时（例如：没有算子被提议、多个算子偏好相同、算子的应用结果不确定），就会产生僵局。

SOAR 的处理方式是将"解决这个僵局"本身作为一个新的子目标，在一个新的子问题空间中递归地进行搜索。这个机制使得 SOAR 能够：

- 自动将复杂问题分解为子问题（无需预先编程分解策略）
- 在不同知识层次之间灵活切换（从快速的规则匹配到慢速的深度搜索）
- 通过"组块学习"（chunking）将子问题的解决方案编译为新的产生式规则，加速未来类似情境的处理

### 知识表示

SOAR 使用产生式规则作为统一的知识表示形式：

```
# SOAR 产生式规则示例
sp {propose*move-to-destination
    (state <s> ^name navigate ^destination <dest> ^current-location <loc>)
    (<loc> ^connected <dest>)
-->
    (<s> ^operator <op> +)
    (<op> ^name move ^destination <dest>)
}
```

SOAR 的工作记忆是一个图结构（类似语义网络），产生式规则在其上进行模式匹配和状态修改。

### 学习机制

SOAR 具有多种学习机制：

- **组块学习（Chunking）**：将子目标的求解过程压缩为一条新规则，使得下次遇到类似僵局时可以直接响应
- **强化学习**：通过奖励信号调整算子选择的偏好值
- **语义记忆和情节记忆**：长期存储事实知识和经历片段

## ACT-R (Adaptive Control of Thought—Rational)

### 历史背景

ACT-R 由 John Robert Anderson 在 1976 年于卡内基梅隆大学提出初版（ACT），经过 HAM（1973）→ ACT（1976）→ ACT*（1983）→ ACT-R（1993）的持续演进。ACT-R 的"R"代表"Rational"，强调人类认知是在计算资源约束下对环境的理性适应。

### 核心架构

ACT-R 区分两种根本不同的知识类型：

**陈述性知识（Declarative Knowledge）**：存储在"块"（chunks）中，表示事实和经验。每个块有一个类型（chunk-type）和若干槽位（slots），类似于结构化记录。

```
;; ACT-R chunk 示例
(chunk-type addition-fact addend1 addend2 sum)
(3+4  ISA addition-fact  addend1 3  addend2 4  sum 7)
```

**程序性知识（Procedural Knowledge）**：存储在产生式规则（production rules）中，表示"如果...则..."的条件-行动对。产生式规则操作在各模块的缓冲区上。

### 模块化认知架构

ACT-R 的一个核心特征是其模块化设计——不同的认知功能由独立的模块负责，它们通过有限容量的缓冲区（buffers）与中央产生式系统通信：

| 模块 | 功能 | 对应脑区 |
|------|------|---------|
| 视觉模块 | 视觉感知与注意 | 枕叶/顶叶 |
| 听觉模块 | 听觉感知 | 颞叶 |
| 手动模块 | 运动控制 | 运动皮层 |
| 陈述性记忆 | 事实存储与检索 | 颞叶/海马 |
| 目标模块 | 当前目标/意图 | 前额叶 |
| 想象模块 | 问题表示维护 | 顶叶 |

每个模块一次只能在其缓冲区中持有一个块，这个"容量限制"是 ACT-R 模拟人类认知瓶颈（如注意力限制、工作记忆容量）的关键机制。

### 激活与检索

ACT-R 中陈述性知识的检索遵循"理性分析"（rational analysis）原则——系统计算每个记忆块的**激活值**（activation），激活值越高的块越容易被检索到：

```
A_i = B_i + ∑ W_j × S_ji + ε

其中：
- B_i = 基础激活（反映使用频率和新近度的幂律衰减）
- W_j × S_ji = 扩散激活（来自当前注意焦点的关联强度）
- ε = 随机噪声
```

这个公式能够定量预测人类的记忆检索时间和错误率，是 ACT-R 最成功的定量模型之一。

### 学习与适应

ACT-R 的学习机制包括：

- **激活学习**：通过使用频率和新近度自动调整块的基础激活值
- **关联学习**：通过共现统计自动调整块之间的关联强度
- **产生式编译**：将两条连续触发的产生式规则合并为一条，加速后续执行
- **效用学习**：通过奖励信号调整产生式规则的效用值（类似强化学习）

## SOAR 与 ACT-R 的对比

| 维度 | SOAR | ACT-R |
|------|------|-------|
| 核心目标 | 通用问题求解 | 定量认知建模 |
| 知识表示 | 统一产生式规则 | 陈述性+程序性双系统 |
| 学习核心 | 组块化（经验压缩） | 激活调整（统计适应） |
| 验证方式 | 行为复杂度匹配 | 反应时间/准确率定量拟合 |
| 关键创新 | 僵局驱动的子目标 | 理性分析的激活检索 |
| 工程化程度 | 较高（已用于游戏AI、军事模拟） | 中等（主要用于认知实验建模） |
| 并行性 | 规则匹配并行，执行串行 | 模块并行，缓冲区串行 |

两者的根本分歧在于对"认知的本质"的理解：SOAR 将认知视为"搜索"过程，ACT-R 将认知视为"统计适应"过程。

## 在现代 LLM Agent 中的理论指导

### SOAR 对 Agent 设计的启示

**僵局驱动的自动子目标生成：** 当 LLM Agent 在执行任务时"卡住"（不知道下一步做什么、生成了矛盾的方案），可以借鉴 SOAR 的僵局机制——将"解决当前困境"本身作为一个新的子任务，递归调用推理模块。这为 Agent 的"自我反思"和"自动任务分解"提供了理论依据。

**经验组块化与 Few-Shot 学习：** SOAR 的组块学习机制启示我们，Agent 应该能够将成功的问题解决经历"压缩"为可复用的模式。这与 Agent 系统中将成功的 tool-use 序列缓存为"宏动作"（macro-action）的实践不谋而合。

**问题空间的层次化搜索：** SOAR 的多层问题空间启示了 Agent 的规划系统设计——高层空间负责战略决策（选择哪类方法），低层空间负责战术执行（具体参数选择），不同层次可以使用不同的搜索策略。

### ACT-R 对 Agent 设计的启示

**激活式记忆检索：** ACT-R 的激活模型为 Agent 的记忆系统设计提供了优雅的理论基础。向量检索中的"相似度 + 时间衰减 + 重要性加权"组合，本质上就是 ACT-R 激活公式在 LLM 时代的工程实现。

**认知瓶颈的有意设计：** ACT-R 中缓冲区的容量限制并非缺陷，而是特性——它迫使系统聚焦于最相关的信息，避免了组合爆炸。类似地，LLM 的上下文窗口限制可以被重新理解为一种"认知聚焦"机制，关键在于如何智能地选择放入窗口的信息。

**双系统知识架构：** ACT-R 区分"知道什么"（陈述性）和"知道怎么做"（程序性）的设计，启示了 Agent 系统中"知识库"与"技能库"的分离。RAG 检索的是陈述性知识，而 tool-use 和 workflow 执行的是程序性知识——两者需要不同的存储和检索机制。

### 融合设计：BDI + SOAR + LLM

2024-2026 年的实践表明，一种强大的 Agent 设计模式正在形成——以 BDI 作为顶层架构指导战略意图，以 LLM 作为知识源和"直觉引擎"，以 SOAR/ACT-R 思想作为结构化推理子系统：

```
┌──────────────────────────────────────────────────┐
│  BDI 层：战略意图管理                              │
│  - 愿望：长期目标（如"完成代码重构"）              │
│  - 意图：当前承诺的计划                            │
│  - 审议：选择下一个要追求的目标                    │
├──────────────────────────────────────────────────┤
│  LLM 层：知识与直觉                               │
│  - 信念生成：理解环境状态和约束                    │
│  - 选项生成：创造性地提出候选方案                  │
│  - 自然语言理解与生成                             │
├──────────────────────────────────────────────────┤
│  SOAR-like 推理层：结构化问题求解                  │
│  - 计划分解：将意图分解为可执行步骤                │
│  - 僵局处理：遇到阻碍时递归分解                    │
│  - 经验缓存：将成功路径组块化为宏动作              │
├──────────────────────────────────────────────────┤
│  ACT-R-like 记忆层：智能检索                       │
│  - 激活式检索：频率 × 新近度 × 关联度             │
│  - 容量管理：聚焦最相关的上下文                    │
│  - 双系统存储：知识库 + 技能库                     │
└──────────────────────────────────────────────────┘
```

## 历史评价

SOAR 和 ACT-R 的最大贡献在于证明了"通用认知机制"的可计算性——复杂的智能行为可以从少数基本机制的交互中涌现。它们数十年积累的认知实验数据和理论洞察，为 LLM 时代的 Agent 设计提供了珍贵的"人类智能参考实现"。

然而，它们的局限也很明显：知识获取瓶颈（需要人工编码大量规则）、缺乏处理自然语言和非结构化信息的能力、难以扩展到开放域任务。这些恰好是 LLM 的长处——这也解释了为什么认知架构理论 + LLM 能力的融合成为 2025-2026 年的重要研究方向。
